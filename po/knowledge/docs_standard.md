# Documentation Standards: Product Owner

As a Product Owner, your documentation focuses on business logic, use cases, and requirements.

1. **Format**: Use the Logseq outliner format (bulleted lists).
2. **Core File**: `docs/pages/BUSINESS_FLOW.md` is your primary anchor.
3. **Links**: Use `[[double brackets]]` to link to technical concepts, UI flows, and actors.
4. **Context Preservation**: Always read existing content first. Never delete historical context; append or update state flags (e.g., `[DONE]`, `[OBSOLETE]`).
