- # Internal Investigation Protocol
	- **New Shared Skill — `common/skills/investigation.md`**: Added a four-phase read-only behavioral simulation skill to the common skills layer. Phases: Asset Mapping → Data Profiling → Behavioral Simulation → Findings Report. Designed for internal Q&A against local files (CSV, JSON, configs, source code) and hypothetical data scenarios. Zero side effects: no Logseq artifacts, no implementation transitions, no commits. (ref: `common/skills/investigation.md`)
	- **Relevance Filter Update — `index.js`**: Added `investigation.md` to the `compileCommonSection` heuristic guard. Injected only when the search target contains analyze/simulate/hypothetical/trace/csv/json/data file keywords, preventing token bloat and read-only framing on action-oriented commands. (ref: `index.js → compileCommonSection`)

- # New `analyze` Command — Five Agents
	- **`architect:analyze`**: System-level behavioral simulation. Traces infrastructure configs, data flows, and inter-service contracts against real or hypothetical inputs. (ref: `architect/commands/architect/analyze.toml`)
	- **`backend:analyze`**: API and data contract simulation. Traces JSON/CSV payloads through endpoint handlers, validators, ORM queries, and business logic branches. (ref: `backend/commands/backend/analyze.toml`)
	- **`frontend:analyze`**: Component and state simulation. Traces API response shapes through component trees, conditional rendering paths, and state mutations. (ref: `frontend/commands/frontend/analyze.toml`)
	- **`mobile:analyze`**: Dart/Flutter runtime simulation. Traces data against widget rebuild paths, null-safety conditions, BLoC/Riverpod transitions, and async stream behavior. (ref: `mobile/commands/mobile/analyze.toml`)
	- **`po:analyze`**: Business rule simulation. Traces data scenarios against acceptance criteria and business constraints. Opportunistically loads an existing PRD as passive context if found — not a required dependency. (ref: `po/commands/po/analyze.toml`)
