- # Fixed MCP Sampling Loop Early-Exit & SonarQube Unavailability Reporting
	- Addressed an issue in `run_agent_loop` where the server-side loop parser would exit prematurely if the assistant text contained words like "complete", "done", or "resolved". This bug prevented developer agents from performing closure steps like proposing a branch, creating commits, and pushing to origin.
	- Refactored `run_agent_loop` inside [[index.js]] to:
		- Update `completeRegex` to match both `<task_complete>` and `<task_complete summary="..." />` via optional non-capturing group.
		- Remove substring-based matching for "complete", "done", and "resolved".
		- Enforce that the loop only breaks upon matching the explicit `<task_complete>` tag.
	- Addressed an issue where peer reviewers silently suppressed the unavailability of the SonarQube MCP tool.
	- Updated [[common/skills/base_reviewer.md]], [[backend/commands/backend/squad-review.toml]], [[frontend/commands/frontend/squad-review.toml]], and [[mobile/commands/mobile/squad-review.toml]] to require reviewer agents to explicitly log/state in their review findings/report if the SonarQube MCP tool is not available in their execution environment before falling back to manual/static analysis.
	- Updated [[docs/pages/TECHNICAL_SPECS.md]] to document the strict completion rules for the server-side execution loop.
	- Registered the new changes in [[docs/pages/registry.md]].
- # Renamed MCP Server and CLI to tech-agents
	- Renamed the Model Context Protocol (MCP) server name and CLI wrapper from `agent-hub` to `tech-agents`.
	- Renamed `bin/agent-hub.js` to `bin/tech-agents.js` using `git mv` and updated the `"bin"` and script targets in [[package.json]].
	- Updated [[index.js]] to instantiate the server name as `"tech-agents"` and updated connection logs.
	- Updated [[bin/tech-agents.js]] to rename the Commander application and update all printed instructions.
	- Built self-healing migration logic in [[bin/tech-agents.js]]: the `bootstrap` configuration updater now automatically deletes the stale `"agent-hub"` key and registers the new `"tech-agents"` key in all environment configurations (`settings.json`, `mcp_config.json`, and `claude_desktop_config.json`).
	- Renamed the tool prefix namespace from `agent_hub:` to `tech_agents:` across all 26 TOML prompt files using a workspace-wide search and replace.
	- Updated all test suites (`test/mcp-approve-test.js` and `test/mcp-integration.mjs`) and documentation pages (`AGENTS.md`, `GEMINI.md`, `CLAUDE.md`, `README.md`, `docs/pages/TECHNICAL_SPECS.md`, and `docs/pages/registry.md`).
- # Added adhoc consultation command to po, architect, backend, frontend, and mobile
	- Added a non-coding, consultative/brainstorming command `adhoc` (implemented via `adhoc.toml`) under `po`, `architect`, `backend`, `frontend`, and `mobile` agent commands directories.
