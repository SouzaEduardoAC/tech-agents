- # MCP Transport & Handshake Fixes
	- **Fix stdio Transport**: Replaced `stdio: "inherit"` anti-pattern with `stdio: "pipe"` + explicit stream forwarding in the CLI wrapper (`bin/agent-hub.js serve`) to prevent JSON-RPC stream interception. (ref: `bin/agent-hub.js`)
	- **Direct Entry Point**: Updated `bootstrap` to configure MCP clients (`mcp_config.json`, `claude_desktop_config.json`) to invoke `index.js` directly, bypassing the CLI wrapper. (ref: `bin/agent-hub.js`)
	- **Startup Diagnostics**: Wrapped the MCP transport connection in a `try/catch` block that writes fatal errors to `process.stderr` and exits gracefully for visibility to MCP hosts. (ref: `index.js`)
	- **Bootstrap Migration Guard**: Added stale-entry detection to `updateMcpServers()`. The bootstrap process is now fully idempotent, automatically migrating old `bin/agent-hub.js` paths to `index.js`. (ref: `bin/agent-hub.js`)

- # Prompt Completeness & LLM Discoverability
	- **Tool Description Enrichment**: Enhanced descriptions for `list_agents` and `call_agent_command` with natural-language triggers and agent-role hints, enabling LLMs to map intent to specific agent tools seamlessly. (ref: `index.js`)
	- **Agent Identity Auto-Injection**: Replaced fragile `!{cat}` reliance in `call_agent_command` with `readAgentDirDeduped()`. It automatically loads all `skills/` and `knowledge/` for an agent, skipping files already explicitly `!{cat}`'d in the TOML to ensure full identity loading without token duplication. (ref: `index.js`)
	- **Noise Gating**: Added a heuristic filter to `compileCommonSection` that gates `business_synthesis.md` behind explicit export/synthesis keywords, protecting debate-centric commands (e.g., `council:debate`) from irrelevant formats. (ref: `index.js`)

- # Monorepo Stack Detection
	- **Depth-1 Workspace Scan**: Fixed silent stack-detection failures in multi-module workspaces. Replaced flat `process.cwd()` checks with `scanWorkspace()` to scan both the root and immediate subdirectories (skipping `.git`, `node_modules`). (ref: `index.js`)
	- **Module Attribution**: Enriched the "On-Demand Manifest" (multi-stack mode) to output explicit module paths (e.g., `Java → module-auth/`, `React → module-frontend/`), preventing cross-contamination in monorepos. (ref: `index.js`)

- # Testing Infrastructure
	- **Integration Test Suite**: Created a robust `test/mcp-integration.mjs` test runner that validates the raw stdio transport. Added 36 assertions including:
		- **Content Fingerprints**: Ensures `!{cat}` and auto-injected files actually resolve in the prompt.
		- **Noise Guards**: Verifies isolated commands don't leak `business_synthesis.md`.
		- **Depth-1 Simulation**: Simulates a monorepo workspace to guarantee the dynamic stack detector correctly maps modules to technology stacks.
