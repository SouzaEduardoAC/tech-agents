- # Default MCP Restructuring & Context7 Planning Enforcement
	- **MCP Stack Reorganization**: Decoupled `Playwright MCP and Playwright CLI` from default, out-of-the-box bootstrapping, shifting it to recommended/optional external servers to maintain a lightweight, project-scoped core stack. (ref: `[[TECHNICAL_SPECS]]`)
	- **Context7 Default Integration**: Promoted Upstash's **Context7 MCP** (`@upstash/context7-mcp`) to the default bootstrap server list. Added `CONTEXT7_API_KEY` placeholders inside the local environment configuration (`~/.gemini/settings.json`) to eliminate cut-off hallucinations via live API documentation indexing. (ref: `[[TECHNICAL_SPECS]]`, `bin/agent-hub.js`)
	- **Planning-Only Directive**: Codified the **Context7 Documentation Directive** in `common/knowledge/anti_hallucination.md` to restrict tool calls to planning (research/architecting) levels, preserving tokens during pure execution. (ref: `common/knowledge/anti_hallucination.md`)
	- **Automated Citation Validation**: Synchronized all four core implementation agent protocols (`architect`, `backend`, `frontend`, `mobile`) with a strict automated verification check inside `[MODE: PLAN]`. Agents are now structurally required to capture live API/library definitions under a dedicated plan section (`## Dependency & API Contracts (Context7)`) and verify its presence before human gate reviews. (ref: `architect/skills/protocol.md`, `backend/skills/protocol.md`, `frontend/skills/protocol.md`, `mobile/skills/protocol.md`)
