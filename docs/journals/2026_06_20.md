- # Feature: Cross-Project Pipeline State Isolation & Gitignore Auto-Appending
	- **Problem**: The pipeline state file `.squad-state.json` was statically resolved to the `agent-hub` repository root. Running pipeline sessions in multiple different projects caused state collisions and overwrites. Furthermore, `.squad-state.json` was left as an untracked file in target projects, polluting git workspaces.
	- **Dynamic Resolution**: Updated `index.js` to search upward from `process.cwd()` to dynamically resolve `.squad-state.json` or find project root markers (`.git`, `package.json`).
	- **Gitignore Auto-Appending**: Integrated automatic `.gitignore` modification into `pipeline_start`. If a `.gitignore` exists at the resolved project root, `.squad-state.json` is appended if not already present.
	- **Command Template Refactor**: Modified `squad/commands/squad/approve.toml` to replace the absolute hardcoded path with a relative reference.
	- **Documentation Sync**: Updated version to `1.9.8` in `package.json`, generated log records in `CHANGELOG.md`, `docs/pages/registry.md`, `docs/pages/TECHNICAL_SPECS.md`, and `docs/pages/Squad Orchestrator.md`. (ref: `index.js`, `squad/commands/squad/approve.toml`, `package.json`)
- # Feature: MCP Approval Parity & Repository Containment
	- **Problem**: There was no standard MCP tool to register human approvals directly via tool invocation, forcing users to type slash commands. In addition, when executing commands in target projects (outside the agent hub), the documentation sync commands (`full-sync`, `docs`) would run incorrectly, and agents would attempt to use Logseq-specific outliner formats even when no `docs/pages/` folder existed, leading to formatting errors.
	- **MCP Approval Tool**: Added the `pipeline_approve` MCP tool to ListToolsRequestSchema and CallToolRequestSchema in `index.js`.
	- **Pause Instructions Update**: Updated `request_approval` and `check_gate` pause instructions to guide humans that they can approve either in chat or by calling `pipeline_approve`.
	- **Repository Containment & Adaptation**: Implemented `isInsideHub()` repository check. Instead of blocking documentation commands (`full-sync`, `docs`) outside the Hub, the server dynamically injects an `EXTERNAL WORKSPACE NOTICE` instructing agents to avoid modifying Hub anchors (`AGENTS.md`, `GEMINI.md`, `CLAUDE.md`) and focus on the external project's own files.
	- **Compliance Mandate**: Appended the compliance mandate from `AGENTS.md` to the agent prompt when inside the Hub workspace.
	- **Logseq Warning**: Injected a warning into agent prompts if `docs/pages/` is missing, instructing them to write standard markdown instead of Logseq outliner style.
	- **Verification**: Created `test/mcp-approve-test.js` verifying all 13 test scenarios for inside and outside the hub.
	- **Documentation Update**: Documented the new tool in `CLAUDE.md`, `README.md`, `docs/pages/registry.md`, and this journal. (ref: `index.js`, `test/mcp-approve-test.js`, `README.md`, `CLAUDE.md`)
